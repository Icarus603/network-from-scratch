## 核心概念

### 1. Design section 結構

```text
3. Design

3.1 Overview (with Figure 1 architecture diagram)
3.2 Threat model formalization (carry from §2; or reference)
3.3 Cryptographic core
    3.3.1 Handshake (hybrid X25519+ML-KEM)
    3.3.2 Record layer
    3.3.3 Forward secrecy / key update
3.4 Active probing resistance
    3.4.1 Fallback path zero-copy
    3.4.2 Timing equalization invariant
3.5 Adaptive shaping
    3.5.1 Profile model
    3.5.2 Pacing & padding
3.6 Formal verification
    3.6.1 ProVerif model
    3.6.2 TLA+ state machine
3.7 Design rationale / alternative considered (1 paragraph)
```

每 sub-section 之 pattern:

```
[Goal sentence] In this subsection, we describe X.
[Problem sentence] The fundamental difficulty in X is ...
[Solution sentence] Our approach is ...
[Detail paragraphs] Concretely, ...
[Figure / formula]
[Why this works] We argue this satisfies Goal G_i because ...
```

對 reviewer 之 expectation：**precise, not exhaustive**。Spec full details 放 appendix；body 只 «enough to convince + understand».

### 2. Implementation section

```text
4. Implementation

[Para 1] Code size + language + lines
[Para 2] Architecture choices (Rust core + Go shim — refer to §3.1)
[Para 3] Performance-critical paths (io_uring, AEAD in-place, FFI batching)
[Para 4] Open-source artifact pointer

Table 1: Lines of code per component
```

reviewer 想看 «是真做了還是只設計»。實作數字（LoC, dependencies, CI passing rate）是 evidence.

### 4. Evaluation 之 writing pattern

每 sub-section 之 standard pattern:

```
[Research question] RQ_n: «Does protoxx achieve goal G_i?»
[Method] We tested X by Y. Setup: ... Baseline: ...
[Result] Figure N / Table M.
[Interpretation] We observe that ... This means ...
[Comparison] vs baseline ... protoxx achieves X% improvement.
[Limitation] However, we caution ...
```

範例:

```text
5.2.1 Throughput

RQ1: Can protoxx match or exceed Hysteria2's throughput in 
representative scenarios?

We benchmark protoxx and five baseline protocols (Hysteria2 
v2.6, TUIC v5, VLESS+REALITY, shadowsocks-2022, WireGuard) 
on a 25 Gbps testbed (described in §5.1) under three network 
profiles: LAN (0 RTT, 0% loss), WAN-clean (30 ms RTT, 0% loss), 
and WAN-loss (30 ms RTT, 1% loss). Each datapoint is the mean 
of 5 60-second iperf3 runs; error bars are 95% bootstrap CI.

Figure 4 shows throughput vs concurrency. Protoxx achieves 
10.4 ± 0.3 Gbps single-stream on LAN, surpassing Hysteria2's 
8.2 ± 0.4 Gbps by 27% (p < 0.001, Welch's t-test). Under 
WAN-loss, protoxx with adaptive FEC achieves 2.7 ± 0.1 Gbps, 
vs Hysteria2 brutal at 2.1 ± 0.2 Gbps and TUIC at 1.4 ± 0.1 
Gbps. We attribute the LAN advantage to io_uring batching 
(Section 4) and the WAN advantage to adaptive FEC trigger 
above 0.5% loss.

Caveat: We measured on a single hardware configuration. 
Smaller VPS (1-2 cores) likely reduce the LAN advantage.
```

關鍵：
- 數字 + CI
- statistical test 註明
- 對比 baseline 明確
- attribute / explain
- caveat 主動 surface

### 5. Figure design

對 evaluation 主圖（throughput / classifier accuracy / probe resistance）：

```text
- 2-column width (≈ 250pt) 或 full-width (≈ 540pt)
- our protocol 用 highlight color (red/orange) + bold
- baselines 用 grey/blue tonal
- log-scale 對 latency / loss / throughput 之 wide range
- error bars 必須
- legend 對 paper 之 §reference (e.g. «protoxx (§3, ours)»)
- caption 自含 setup + sample size
```

### 6. Formal verification 之 reporting

```text
We model protoxx's handshake in ProVerif under the Dolev-Yao 
adversary. The model encodes G1 (confidentiality), G2 (mutual 
authentication), G3 (forward secrecy), and G4 (KCI resistance) 
as ProVerif queries:

  query attacker(session_key).
  query inj-event(server_accept) ==> inj-event(client_init).
  ...

ProVerif terminates and proves all four queries hold under 
honest-server, compromised-client and 
compromised-server-after-session cases. Listing 1 shows the 
ProVerif process for the client. Full models are in 
[github.com/...] and verification logs in Appendix A.

We additionally model the protocol state machine in TLA+ to 
verify safety invariants (no double-decrypt, no key reuse) 
and liveness (handshake completes within 3 round trips). 
TLC model-checked the spec at depth 12 with no counterexample.
```

reviewer 對 formal 之 expectation：方法 explicit + tool version + artifact + bounded model size 明示.

### 7. Real-world result 之 reporting

reviewer 對 measurement-style result 之最 sensitive：

```text
Real-world deployment: we deployed protoxx on a VPS in 
Tokyo and a measurement client in mainland China (Shanghai 
Aliyun VPS, registered with required ID under PRC regulations 
[reference]). Over 30 days (2026-01-01 — 2026-01-31), 
client initiated a 1-MB transfer to server every 30 minutes 
(total 1,440 sessions). 

Table 4 reports per-day median throughput, success rate, and 
GFW probe count against four servers (protoxx, vanilla Caddy, 
VLESS+REALITY, Hysteria2). Vanilla Caddy serves as control 
for non-protocol confounders (IP block, ISP routing).

Result: protoxx achieved 99.2% session success rate, comparable 
to vanilla Caddy (99.8%) and exceeding VLESS+REALITY (94.3%) 
and Hysteria2 (78.1%). Over the 30 days, protoxx server received 
2.3 probes/day from non-Tokyo IPs (vs Caddy 1.8/day, 
VLESS+REALITY 7.4/day, Hysteria2 31.2/day), suggesting GFW 
does not differentiate protoxx from cover web traffic.

Caveats: (1) measurement uses single ISP (Aliyun); GFW behavior 
varies by ISP/region. (2) 30 days is short; long-term policy 
changes could alter results. (3) Our experiment volunteered 
no third-party user; opt-in beta deployment is future work.
```

### 8. Discussion section

```
6. Discussion

[Para 1] Trade-offs: throughput-stealth Pareto curve
[Para 2] Limitations of evaluation: lab-to-prod gap, single ISP
[Para 3] Threats to validity (internal, external, construct)
[Para 4] Negative result / surprise (we found X 反 prediction)
[Para 5] Future work: PQ-only handshake, multipath
```

reviewer 給 high credit for «honest discussion» — 不是只羅列 limitation，要 explain implication.

### 9. 表達 trade-off 之 figure

```mermaid
flowchart LR
    P[Pareto plot] --> AX[x: detection rate (lower better)]
    P --> AY[y: throughput / CPU]
    P --> M[Multiple protocols dotted; ours highlighted]
```

paper 中通常 1 個 prominent figure 展示 «our improves frontier»。

### 10. Statistical rigor checklist

```text
[Q-EVAL-1]  每 datapoint 有 ≥ 5 repeat
[Q-EVAL-2]  Error bar 用 95% CI（標明 bootstrap or normal）
[Q-EVAL-3]  比較 用 statistical test（t / Mann-Whitney / KS）
[Q-EVAL-4]  Effect size 明示（Cohen's d / 比例）
[Q-EVAL-5]  Multiple testing correction（Bonferroni）若 ≥ 5 對比
[Q-EVAL-6]  Variance source 明示（intra-run, inter-run, inter-host）
[Q-EVAL-7]  Cherry-picking 顯式 reject：show worst case too
[Q-EVAL-8]  Reproducibility recipe in artifact section
```

### 11. 字數預算分配

對 13-page paper：
- Abstract: 0.15 page
- Intro: 1.2 page (§1)
- Background + Threat: 1.5 page (§2)
- Design: 4 page (§3) — 最大
- Implementation: 1 page (§4)
- Evaluation: 4 page (§5) — 第二大
- Discussion: 0.8 page (§6)
- Related: 0.8 page (§7)
- Conclusion: 0.2 page (§8)
- References: 1 page

Tight。每 page 都要 deliver content；無 filler.

### 12. Camera-ready 與 artifact

接收後：
- shepherd 監督 revision (2-4 weeks)
- artifact evaluation 同時跑（USENIX AE 流程）
  - submit Docker image / repo + 重現 script
  - AE 三層 badge: «Available», «Functional», «Reproduced»
- 最後 camera-ready PDF 上載

USENIX AE Reproduced badge 對 paper 之 visibility 顯著加分。

## 研究級補遺

### 1. 學界詞彙

| 中文/口語 | 學界詞彙 |
|---|---|
| 研究問題 | research question (RQ) |
| 內部有效性 | internal validity |
| 外部有效性 | external validity / generalizability |
| 構念有效性 | construct validity |
| 效應大小 | effect size |
| 統計顯著 | statistical significance |
| 重現性徽章 | reproducibility / availability / functional / reproduced badge |

### 2. 對手分類學

reviewer 角度的「對手」：

| Reviewer concern | Defense |
|---|---|
| «evaluation 不 rigorous» | CI + repeat + statistical test |
| «沒比 SOTA» | comprehensive baseline matrix |
| «cherry-pick» | show worst case + Pareto frontier |
| «no artifact» | reproducible repo + AE submission |
| «over-claim» | precise wording + concrete numbers |
| «under-deliver» | match abstract promise in §5 |

### 3. 形式化定義

**Internal validity**: 對於我們 evaluation, $\Pr[\text{conclusion} | \text{observation}]$ 高 — 控制 confounder。
**External validity**: 結論在 unseen 環境的 generalization。
**Effect size**: $d = (\mu_1 - \mu_2) / \sigma_{\text{pool}}$.

### 7. 開放問題

1. **Evaluation 之 «enough» 邊界**：reviewer 對 evaluation 之 demand 在不同 venue 有 variance — open empirical
2. **Real-world result 之 statistical methodology**：30-day window 之 representativeness — open
3. **Formal claim 之 confidence 量化**：ProVerif 之 «proves» 對 reader 之信任 — 仍 dependent on tool trust
4. **Long-term impact metric**：paper 對 community 之 lasting impact 之 quantifiable proxy — meta-research direction
