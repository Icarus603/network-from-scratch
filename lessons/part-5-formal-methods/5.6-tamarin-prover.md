## 動機

ProVerif 對純 Dolev-Yao secrecy + simple authentication 又快又好。但對需要：
- **DH algebraic property** 推理（多 exponentiation, group structure）
- **Mutable global state** (anti-replay, PKI revocation)
- **Multi-stage key exchange** (TLS 1.3 三 epoch keys 之間 independence)

ProVerif 處理困難。**Tamarin** 是這些場景的 default tool。

讀完應該能：
1. 看懂 Tamarin multiset rewriting rule
2. 理解 backwards reasoning 跟 ProVerif Horn clause 的差異
3. 寫 simple protocol Tamarin spec
4. 對 TLS 1.3 / WireGuard Tamarin model 讀懂結構

## 核心概念

### 1. Tamarin 的根：multiset rewriting

protocol 在 Tamarin 是 **labeled multiset rewriting system**：
- **State** = multiset of **facts**
- **Transition** = **rule** `[premise_facts] --[ action_facts ]--> [conclusion_facts]`
- **Actions** 留 trace 供 property reasoning

#### Fact 語法
```
Fact_name(arg1, arg2, ...)
```

幾種 reserved fact：
- `Fr(x)`: x 是 fresh nonce（系統 builtin)
- `In(m)`: 從 network 收到 message m（attacker provides）
- `Out(m)`: 送到 network message m（attacker observes/intercepts）
- `K(m)`: attacker knows m
- `!Fact(...)`: persistent fact（不消耗）；無 `!` prefix = linear fact（消耗）

#### Rule 範例

```tamarin
rule Send_msg:
    [ Fr(~n) ]                          // premise: fresh nonce
  --[ Send(~n) ]->                       // action: log
    [ Out(<'msg', ~n>) ]                 // conclusion: put on network
```

讀作：「如果有 fresh nonce ~n，可以送 `<'msg', n>` 到 network，並 log Send(n) action」。

### 2. 一個完整 simple example

兩個 party 透過 pre-shared key 通訊：

```tamarin
theory PSKExample
begin

builtins: symmetric-encryption

// Setup: A and B 共享 PSK k
rule Setup_PSK:
    [ Fr(~k) ]
  --[ SetupPSK($A, $B, ~k) ]->
    [ !PSK($A, $B, ~k), !PSK($B, $A, ~k) ]

// A sends message m to B
rule A_send:
    [ !PSK($A, $B, k), Fr(~m) ]
  --[ Sent($A, $B, ~m) ]->
    [ Out(senc(~m, k)) ]

// B receives message
rule B_recv:
    [ !PSK($A, $B, k), In(c) ]
  --[ Received($A, $B, sdec(c, k)) ]->
    [ ]

// Properties

lemma message_secret:
    "All A B m #i. Sent(A, B, m) @ #i ==> not Ex #j. K(m) @ #j"

lemma authentication:
    "All A B m #i. Received(A, B, m) @ #i ==>
        Ex #j. Sent(A, B, m) @ #j & #j < #i"

end
```

- `$A`, `$B`: public constants (party names)
- `~k`, `~m`: fresh nonces (note `~` prefix)
- `!PSK(...)`: persistent fact
- `#i`, `#j`: time points (every action has temporal index)
- `@`: action happens at time point

跑 Tamarin：
```bash
tamarin-prover --prove psk.spthy
```

對每個 lemma 試證明。`message_secret` 應 PASS（PSK k 不會 leak）。`authentication` 應 fail：因為 attacker 可以 replay senc(m, k)！需要 nonce + freshness 機制才能達 injective authentication。

### 3. Backwards reasoning

Tamarin 對 lemma 用 **constraint solving**:
1. 對 negation of lemma 構造 constraint system
2. 解 constraint: 如果 found solution → counter-example (attack)；若 no solution after 完整 search → lemma proved

對前面 `message_secret`:
- 假設 `K(m)` 在某時點 j 成立
- 要 m 是 ~k attacker 不知道, m 必須是 Sent action 中那個 ~m
- attacker 看到 `senc(m, k)` 但要 derive m 需要 k
- k 是 !PSK 內 fresh, 從未 Out → attacker 無法 derive
- Constraint solver 找不到 K(m) 的 derivation tree → lemma proved

對 `authentication`:
- 假設 Received(A, B, m) 但無對應 Sent
- attacker 可以從 Out 拿到 senc(m', k) 對某 m'
- 用 PSK k 它 decrypt 得 m'
- 改 replay 給 B → B 收到，認為 m' 是 fresh request

→ Tamarin 找到 attack trace, 印給你。

### 4. Diffie-Hellman in Tamarin

Tamarin builtin DH theory:
```tamarin
builtins: diffie-hellman
```

定義 group + exponentiation + the key equation:
$$\text{exp}(\text{exp}(g, x), y) = \text{exp}(g, x \cdot y)$$

Tamarin 自動知 `exp(exp(g, x), y) = exp(exp(g, y), x)`。

Simple DH:
```tamarin
rule Init_send:
    [ Fr(~x) ]
  --[ ]->
    [ St_A_init(~x), Out(exp('g', ~x)) ]

rule Resp_send:
    [ Fr(~y), In(X) ]
  --[ ]->
    [ St_B_resp(~y, X), Out(exp('g', ~y)) ]

rule Init_finish:
    [ St_A_init(~x), In(Y) ]
  --[ Key_A(exp(Y, ~x)) ]->
    [ ]

rule Resp_finish:
    [ St_B_resp(~y, X) ]
  --[ Key_B(exp(X, ~y)) ]->
    [ ]

lemma key_secret:
    "All k #i. Key_A(k) @ #i ==> not Ex #j. K(k) @ #j"
```

textbook DH 沒 auth → key_secret 對 MITM 不成立。

加 signature (modeling RSA / EdDSA):
```tamarin
builtins: signing

rule Register_pk:
    [ Fr(~sk) ]
  --[ RegisterPubkey($A, pk(~sk)) ]->
    [ !Ltk($A, ~sk), !Pk($A, pk(~sk)) ]

rule Init_send:
    [ Fr(~x), !Ltk($A, skA) ]
  --[ ]->
    [ St_A_init(~x, $A), Out(<exp('g', ~x), sign(exp('g', ~x), skA)>) ]
```

driven by Tamarin's automated DH + signature reasoning, signed-DH 應該 secrecy + authentication PASS.

### 5. Tamarin vs ProVerif: 對比

| 特性 | ProVerif | Tamarin |
|---|---|---|
| 內部表示 | Horn clauses | Multiset rewriting + first-order logic |
| Reasoning | Forward (saturation) | Backwards (goal-directed) |
| DH algebraic | 部分支援 (need user rules) | 內建完整 |
| Stateful global state | awkward (GSVerif extension) | 自然 (linear facts + persistent facts) |
| Termination | usually OK | sometimes problematic, may need lemma hints |
| Speed | 通常更快 | 通常較慢 |
| Output | Yes / No / Cannot decide + trace | Yes / No + trace + interactive |
| Learning curve | Moderate | Steep |
| Best for | Pure Dolev-Yao + secrecy/auth | DH + multi-stage + stateful |

實務：先用 ProVerif 試 secrecy/auth；複雜場景轉 Tamarin。Cremers 2017 TLS 1.3 用 Tamarin 是因為 multi-stage MSKE + DH 結構 ProVerif 處理不好。

### 6. Lemma 寫作技巧

Tamarin lemma 是 first-order logic 加 temporal index. 常見 pattern:

#### Secrecy
```
lemma secret_key:
    "All k #i. SecretKey(k) @ #i ==> not Ex #j. K(k) @ #j"
```

#### Authentication (non-injective)
```
lemma auth_basic:
    "All A B m #i. Received(A, B, m) @ #i ==>
        Ex #j. Sent(A, B, m) @ #j & #j < #i"
```

#### Injective authentication
```
lemma auth_injective:
    "All A B m #i. Received(A, B, m) @ #i ==>
        (Ex #j. Sent(A, B, m) @ #j & #j < #i) &
        not (Ex A2 B2 m2 #i2.
                (Received(A2, B2, m2) @ #i2) &
                (not (#i = #i2)) &
                (Same(m, m2)))"
```

注意 `not (#i = #i2)` 確保唯一性。

#### Forward secrecy
```
lemma forward_secrecy:
    "All k #i #j. SessionKey(k) @ #i &
                  RevealLtk() @ #j & #i < #j ==>
                  not Ex #m. K(k) @ #m"
```

意思：在 long-term key reveal 之前的 session key 即使 ltk 後來洩漏仍 secret.

#### KCI resistance
```
lemma kci_resist:
    "All A B k #i #j. SessionKey(A, B, k) @ #i &
                       RevealLtkOf(A) @ #j ==>
                       not Ex #m. K(k) @ #m"
```

### 7. Helper lemma 與 induction

Tamarin 對複雜 protocol 用「helper lemma」分而治之。例：

```tamarin
lemma helper_x [use_induction, reuse]:
    "All x #i. SomeFact(x) @ #i ==> ..."

lemma main_secret:
    "..."
```

`[use_induction]` 提示 Tamarin 用 induction; `[reuse]` 表示這個 lemma 可在其他 proof 用。

對 TLS 1.3 Cremers 2017 paper 用 50+ helper lemmas 才完成完整證明。

### 8. Tamarin 的 industry track record

| Protocol | Tamarin verify 結果 | 找到的問題 |
|---|---|---|
| **TLS 1.3** | Cremers CCS 2017 完整 | 早期 draft 的 PSK post-handshake auth 漏洞 |
| **5G-AKA** | Basin et al. CCS 2018 | 一些 unlinkability 不成立的 case |
| **WireGuard** | Donenfeld-Milner 2018 | spec ambiguity → fixed |
| **EMV (信用卡)** | Basin et al. USENIX Sec 2021 | 找出 PIN bypass attack on Visa contactless |
| **MLS** | Cremers et al. 持續 | 設計階段 issue → MLS spec 修正 |
| **Signal Double Ratchet** | Cremers et al. (forthcoming) | property 大部分 hold |

EMV 例子最 striking：Visa 設計 2010 的 contactless protocol, Tamarin 2020 找出可以 bypass PIN attack (用 iPhone NFC reader replay payment authorization)。

### 9. Tamarin on TLS 1.3 — Cremers CCS 2017 details

paper Section 4 列 8 condition:
1. **secret session keys** (multiple, per epoch)
2. **mutual authentication**
3. **key independence**: handshake_key 不洩漏 application_key
4. **perfect forward secrecy**
5. **key compromise impersonation resistance**
6. **non-injective authentication**
7. **session uniqueness**
8. **unique session key derivation**

對 draft-21 TLS 1.3 Tamarin verified 大部分 PASS，但發現某些 PSK post-handshake auth 場景 not injective — 推動 spec 修正。

### 10. WireGuard on Tamarin — Donenfeld-Milner 2018

Section 7 列 verified properties:
- Correctness (handshake completes)
- Mutual authentication
- Forward secrecy
- KCI resistance
- Identity hiding (responder)
- Authentication of timestamp (anti-replay)
- Non-malleability
- Resistance to replay across sessions
- Resistance to unknown key-share attack

每個都 Tamarin lemma 形式 + sufficient helper lemmas。**WireGuard.spthy** 約 600 行（含 helpers）。

### 11. Tamarin 的 limitation

- **Non-termination**: 某些 protocol Tamarin 跑不完, 需 interactive mode 給 hint
- **Heavy memory**: large model 可能要 32GB+ RAM
- **Equational theory 限制**: 不能任意 add equation; 內建 AC, XOR, DH, multiset, hash, signature, etc., 不在這個 set 內的 unsupported
- **Probabilistic / computational 不支援**
- **No traffic-level reasoning**: 不能 model packet size / timing

對 anti-fingerprint 的限制：Tamarin 看不到 packet shape；需要 statistical model 補充 (Part 10/11)。

Part 11.10 我們協議 Tamarin model 覆蓋的 lemma:

1. **Inner secrecy**: inner application key 對 attacker (含 long-term ltk reveal scenarios) 不可知
2. **Mutual authentication**: handshake 後雙方 inj-event
3. **Forward secrecy**: long-term key 洩漏不影響 past sessions
4. **KCI resistance** for both sides
5. **Identity hiding** of inner SNI (透過 ECH-style outer)
6. **Multi-stage key independence**: outer key, handshake key, inner key 互相 independent
7. **Anti-replay**: 0-RTT data 不被 accept 兩次（with helper lemma on counter）

## 研究級補遺

### 1. 學界詞彙

| 口語 | 學界用詞 |
|---|---|
| 「Multiset rewriting」 | **Multiset rewriting system (MSR)** |
| 「等式推理」 | **Unification modulo theory** |
| 「Backwards reasoning」 | **Goal-directed search / constraint solving** |
| 「Linear fact」 | **Consumed/produced fact** (linear logic 對應) |
| 「Helper lemma」 | **Auxiliary lemma / intermediate invariant** |
| 「Interactive proof」 | **Interactive theorem proving (with backtracking)** |

### 2. 對手分類學

Tamarin 標準 Dolev-Yao adversary，但 model 可以加：
- `Reveal(A)` action: reveal long-term key of A
- `RevealEphemeral(A)`: reveal ephemeral
- `Compromise(B)`: total compromise

對應 KCI / FS / PFS 不同 lemma 條件。

### 3. 形式化定義

**Tamarin transition relation**: state $S$ → $S'$ via rule $r = (P, A, C)$ iff $P \subseteq S$ (multiset inclusion), $S' = (S \setminus P) \cup C$, with actions $A$ logged.

**Property** = trace formula in first-order logic with temporal indices.

**Soundness**: 若 Tamarin prove property, 則 property 在 protocol 對 attacker 成立。

**Completeness (relative)**: 對特定 theory class (DH, XOR), Tamarin 是完整的 (找得到 attack if any)；對 general theory 是 semi-completion only.
