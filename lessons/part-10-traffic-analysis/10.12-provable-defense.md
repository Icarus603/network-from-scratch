## 動機

10.1 給了資訊理論框架；10.5/10.11 給了 mechanism inventory。本堂面對最硬的問題：**能否「證明」一個 defense 對某類 adversary 安全？**

回答：**部分能。** 給出 4 個 framework：

1. **Wang 2014 USENIX Sec**："provable defense" 第一次出現——對 hand-crafted feature 對手有 attack accuracy upper bound。但 bound 鬆，且只對 specific feature set。
2. **Cherubin 2017 PoPETs**：Bayes-optimal classifier accuracy upper bound——對 **任何** classifier（含 DL）的 upper bound。**這是目前最 general 的 「provability」**。
3. **Differential privacy framework**：traffic version of $(\varepsilon, \delta)$-unobservability。實務上 $\varepsilon$ 巨大，meaningfulness 待商榷。
4. **Randomized smoothing**（視覺領域 borrowed）：給 specific classifier robust radius——但 traffic 上 perturbation set 非 L_p ball, 直接搬不通。

本堂評估這 4 framework 並 sketch G6 計畫採用哪些。

## 核心概念

### 一、What does「provable」mean？

警告：「provable」在 systems literature 中**極度被誤用**。**真實意義必須精確指定**：

- **Provable against 什麼 adversary?** (Bayes-optimal? Specific classifier? DL with bounded compute?)
- **Provable bound 是什麼?** (Upper bound on accuracy? Mutual info?)
- **Provable in 什麼 model?** (asymptotic? finite-sample?)
- **Provable under 什麼 assumption?** (IID traces? Closed-world? Specific feature set?)

Wang 14 的 "provable" = **「對 specific hand-crafted feature classifier，攻擊 acc ≤ specific bound」**。**不蘊含對 DL classifier 也 ≤**。實際上 DF 把 Wang 14 defense 的 accuracy 從聲稱 11% 拉到 80%+。**所以 Wang 14 的 「provable」 在 modern 場景下 misleading。**

### 二、Wang 2014 USENIX Sec 的 provable defense

#### 設計：BuFLO-like + fixed cell quanta

```
For each connection:
  - Send packets at fixed rate ρ
  - Pad total packet count to next multiple of L
  - Total duration = max(T_min, actual_duration_rounded)
```

#### Proof sketch (本論文 Sec 5)

對 attacker classifier $f$ 用 specific feature set $F$（packet count、size sum、duration），constrained class $\mathcal{C}_F$：

$$\text{acc}_f \leq \frac{1}{N} + \frac{|\mathcal{F}_{\text{distinguishable}}|}{N}$$

where $|\mathcal{F}_{\text{distinguishable}}|$ 是 defense 後 feature values 可區分的 site 數。設計 quanta $L$ 大 → 多 site 落在同 quantum → $|\mathcal{F}_{\text{distinguishable}}|$ 小。

#### Caveat

- Bound 對 **feature set $F$** 是緊的，對 outside $F$（如 burst pattern, IAT detail）不適用。
- Sirinam 18 DF 用 raw direction sequence 學到 Wang 14 沒覆蓋的 features，**bound 不再 hold**。
- **教訓：「provable against specific feature」≠「provable in general」**。

### 三、Cherubin 2017 PoPETs："Bayes, not Naïve"

#### Insight

**對任何 classifier**，accuracy 都不能超過 **Bayes-optimal classifier** accuracy。後者 closed-form:

$$\text{acc}^* = E_{y \sim P(Y)}\left[\max_x P(X = x | Y = y)\right]$$

如果能 estimate $\text{acc}^*$，即得 **所有 classifier 的 upper bound**。**這是 DL-proof 的關鍵。**

#### Estimation

對 each test trace $y$，用 kernel density estimator $\hat{P}(X | y)$，再取 max。

- 對 numeric feature space（如 CUMUL 100-dim）可行。
- 對 discrete sequence (raw direction) 需 discretize 或用 embedding。

#### 結果

對 Wang 14 dataset (no defense)：$\text{acc}^* \approx 96\%$. DF 達 98% **顯示 estimator 略 underestimate**（finite sample bias）；理論上 DF $\leq 96\%$。

對 BuFLO defense：$\text{acc}^* \approx 35\%$. DF 60%——bound 仍 valid 但 actual DF 接近 bound。

**Use case**：給 defense 設計者一個 **「攻擊 acc 不可能低於這個」** 的 lower bound on attacker capability，幫助評估 defense effectiveness。

#### G6 用法

對 G6 evaluation，**report Bayes-optimal bound $\text{acc}^*$** —— 不論未來什麼新 attack 出來，都 $\leq \text{acc}^*$. **比單純報 「對 DF 多少 accuracy」 強很多**。

#### Caveats

- $\text{acc}^*$ estimation 有 sample bias；finite-sample bound 鬆。
- estimator quality 取決 representation choice（KDE on which feature space?）。
- 對 high-dim (sequence) 直接 estimate 不可行——必須 invariant feature mapping。

### 四、Differential privacy 框架在 traffic 上

#### $(\varepsilon, \delta)$-unobservability 形式

```
For all adjacent inputs x, x' (differ in one bit of user activity):
  Pr[Defended(x) ∈ S] ≤ e^ε · Pr[Defended(x') ∈ S] + δ
```

對 traffic, **「adjacent」** 怎麼定義？常見定義：

- *Visit-level adjacency*：差異是 user 訪問了哪個 site (e.g., A vs B)。
- *Page-level adjacency*：差異是 user 在 page 內做了什麼動作（click, scroll）。
- *Time-level adjacency*：差異是 user 在某秒是否 active。

不同 adjacency → 不同 DP semantics。

#### 實務問題

對 traffic shaping 直接證 DP 非常難：

1. **Sequential composition**：visit N sites, $\varepsilon$ scales linearly. Tor user 一天看 30 sites → $\varepsilon_{day} = 30 \varepsilon_{visit}$. 若 single $\varepsilon = 1$, daily $\varepsilon = 30$, $e^{30}$ 巨大——無意義 bound。
2. **Sensitivity 不易 bound**：traffic shaping 是 deterministic + randomized injection 混合，敏感度難量化。
3. **Practical**：實作 DP-style padding 需要極大 noise →overhead 爆。

#### Iacovazzi 22 等嘗試

把 DP loosened 為 *expected*-DP 或 *Rényi* DP，給 G6-style mechanism 較緊 bound。**仍處於 academic 探索期**。

### 五、Randomized smoothing 從視覺借鑑

#### 視覺領域定義

> Classifier $g(x) = \arg\max_y P_{z \sim \mathcal{N}(0, \sigma^2)} [f(x + z) = y]$ has certified accuracy radius $r$ around clean $x$.

**Cohen 19 ICML**: $r = \sigma \Phi^{-1}(p_{\text{top}}) / 2$, 其中 $p_{\text{top}}$ 是 smoothed top probability。

#### Traffic 適配難度

- 視覺：perturbation 在 $L_2$ 球；traffic：perturbation 在 「合法 insert / delay」 集合。
- 視覺：input 連續；traffic：input 離散 sequence。
- 視覺：classifier robust 由 smoothing 保證；traffic：defender 是 generator，不是 classifier。

**直接搬不行**。**但 Bahramali 21 TIFS** RATA 嘗試類似 idea，給 robust classifier。**對 defender 角度的 randomized smoothing 尚未有人嘗試** —— 開放研究方向。

### 六、Quantitative Information Flow (QIF) 在 traffic

#### Smith 09 框架

- Gain function $g: \mathcal{X} \times \mathcal{X} \to [0, 1]$，描述 adversary 「贏」 的條件。
- $V_g(X) = E_X [\max_{x'} g(x, x')]$ = vulnerability under $g$.
- $V_g(X | Y) = E_Y [\max_{x'} E_{X|Y} [g(X, x')]]$ = posterior vulnerability.
- $g$-leakage = $\log_2 \frac{V_g(X | Y)}{V_g(X)}$.

#### 對 traffic 應用

```
g(true_site, guess) = 1 if true_site == guess else 0      ← Bayes accuracy
g(true_site, guess) = 1 if rank(guess) ≤ 5 else 0        ← top-5 accuracy
g(true_site, guess) = 1 if both in same "topic" else 0   ← coarse-grain leakage
```

**G6 evaluation 可用多個 $g$**，給 multi-faceted leakage report。**目前無 protocol 這樣做**——G6 開創此 reporting style 是研究貢獻。

### 七、實務「provable」分級

| Level | Description | Example | G6 計畫 |
|---|---|---|---|
| L0 | Empirical evaluation only | most papers | – |
| L1 | Provable against specific feature set | Wang 14 | – |
| L2 | Bayes-optimal upper bound on accuracy | Cherubin 17 | **必有** |
| L3 | DP-style $(\varepsilon, \delta)$-unobservability | none實用 | aspiration, partial |
| L4 | Computational indistinguishability with negligible advantage | none for WF | open problem |

G6 目標：**publish at L2 (Bayes upper bound) + report L3 (DP, partial)**。L4 是長期 research direction。

### 八、Maybenot framework 與 「machine analysis」

Pulls 20 與 Maybenot 系列 (10.5) 提供 padding state machine。**Stuart 24 PoPETs** 提案：對 Maybenot machine 自動估 attacker accuracy upper bound。

- 給 Maybenot machine spec → simulate traces → KDE estimate Bayes-optimal acc。
- 不需要 real implementation 即可 evaluation。

**對 G6**：G6 的 shape layer 寫成 Maybenot machine → 套 Stuart 24 tool 估 leakage bound → **不必跑 actual evaluation 就先 sanity check**。

### 九、Information-theoretic bound 的 fundamental limit

回 10.1 channel capacity 概念。對 G6:

$$C_{G6} = \max_{p(\text{input})} I(\text{site}; \text{wire})$$

若能證 $C_{G6} \leq c_0$ bits/visit, 則：
- 對手 single-visit 攻擊：accuracy 上限 $f(c_0, N)$ via Fano。
- 對手 K-visit aggregation：accuracy 上限增長 $\leq K \cdot c_0$ bits.

**G6 evaluation 必交一個 estimated $C_{G6}$ value**——這是 protocol-level capacity report。

### 十、為什麼這在 SOTA 沒人 publish？

- Tor padding 工程文化 偏向「降 accuracy 多少 percent」, 沒有 formal proof 文化。
- Bayes-optimal estimation 需要 stat training，協議設計者多 systems background。
- 學界激勵：「攻擊 paper 升 accuracy」比「defense paper 證 bound」 容易 publish。

**G6 走「formal report」 路線是 differentiation**——非競爭性，是 community 缺失的 niche。

### 十一、G6 evaluation 的 「provability 四件套」

| Component | Output | Method |
|---|---|---|
| 1. Bayes-optimal acc estimate $\hat{\text{acc}}^*$ | scalar | Cherubin 17 KDE on CUMUL features |
| 2. Channel capacity estimate $\hat{C}$ | bits/visit | Chatzikokolakis 10 estimator |
| 3. DP-form bound $(\varepsilon, \delta)$ | tuple | manual analysis of randomized padding |
| 4. $g$-leakage table | multiple $g$ outputs | Smith 09 QIF for set of gain functions |

**Part 12.20 evaluation paper 必含這四件套**。**這是 G6 「provable defense」 的具體 落地。**

### 十二、Open problems summary

1. **Tighter bound on $C$**：finite-sample Cherubin estimator 鬆，能否 improve?
2. **DP-form sequential composition**：daily visit aggregation 下 $\varepsilon$ 線性 blow-up，能否設計 amortizing scheme?
3. **Randomized smoothing for defender**：能否設計 generator s.t. attacker robust radius 可 certify?
4. **G6 spec 與 ProVerif/Tamarin 結合**：Part 5 form methods 主要 cover crypto safety；traffic shaping 屬於 statistical security——需要新 framework。
5. **Game-theoretic equilibrium for evolving defender/attacker**: 兩方 long-term equilibrium 分析。

## 研究級補遺

### 1. 學界詞彙

- **Provable defense**: defense with formal upper bound on attacker capability
- **Bayes-optimal classifier**: posterior-mode classifier, theoretical best
- **g-leakage**: Smith 09 quantitative info flow
- **Certified accuracy / robustness radius**: Cohen 19 ICML
- **$(\varepsilon, \delta)$-DP**: Dwork-Roth differential privacy
- **Rényi DP**: variant by Mironov 17, supports tighter composition
- **Maybenot machine**: Pulls 20 spec for padding state machine
- **Channel capacity**: Shannon 1948

### 2. 對手分類學

對應 「provability level」 vs adversary classes：

- L0 empirical 只能對 specific attacker (e.g. DF only)
- L1 (Wang 14) covers feature-set $F$ adversary
- L2 (Cherubin 17) covers any Bayes-optimal estimator (含 DL)
- L3 (DP) covers any adversary with bounded query
- L4 covers polynomial-time adversary (未達成)

G6 目標 L2 + partial L3.

### 3. 形式化定義

**Bayes-optimal upper bound**

> For any classifier $f$: $\text{acc}_f \leq \text{acc}^* = E_Y[\max_x P(X=x | Y=y)]$.

**$(\varepsilon, \delta)$-unobservability**

> For all $x_1, x_2$ adjacent: $\Pr[Y \in S | X = x_1] \leq e^\varepsilon \Pr[Y \in S | X = x_2] + \delta$.

**$g$-leakage**

> $L_g(X; Y) = \log_2 \frac{V_g(X | Y)}{V_g(X)}$, with $V_g(X) = E[\max_{x'} g(X, x')]$。
