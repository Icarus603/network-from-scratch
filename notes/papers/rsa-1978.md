# A Method for Obtaining Digital Signatures and Public-Key Cryptosystems
**Venue / Year**: Communications of the ACM, Vol. 21, No. 2, February 1978
**Authors**: Ronald L. Rivest, Adi Shamir, Leonard M. Adleman
**Read on**: 2026-05-14 (in lesson 3.4)
**Status**: full PDF (`assets/papers/rsa-1978.pdf`)
**One-line**: 第一個具體的公鑰加密與簽章方案——把 Diffie-Hellman 1976 的 trapdoor function 概念落地到模 N 冪運算，主導 25 年公鑰密碼學；至 2026 年仍是 90% Web cert 的基礎，但即將被 EdDSA + ML-DSA hybrid 完全取代。

## Problem
Diffie-Hellman 1976 提出 trapdoor function 概念與公鑰加密構想，但**沒有給出具體實作**。三件事仍 open：
1. 是否真的存在「正向算 easy、逆向算 hard」的 trapdoor function？
2. 能用什麼數學結構？
3. 能不能同時實現「加密」與「簽章」？

RSA 1977 設計、1978 publish 的論文回答了上述三個問題。

## Contribution
1. **第一個具體公鑰加密**：基於 RSA Problem (給 (N,e,c) 算 c^d mod N) 與 Factoring Problem。
2. **第一個具體數位簽章**：sk holder 算 σ = m^d mod N，pk holder verify by σ^e mod N == m。注意這是 textbook RSA，現代必加 padding (PSS / FDH)。
3. **數論基礎**：用 Fermat-Euler theorem (a^φ(N) ≡ 1 mod N) 證明正確性；用 factoring 難度當安全性 anchor。
4. **效能 estimate**：1978 年硬體下 200-digit RSA 加密 1 秒、解密 25 秒——當時 hopelessly slow。後 40 年硬體 + algorithm + CRT 加速讓 RSA-2048 sign ms 級。

## Method
```text
KGen:
    Choose two large primes p, q  (random, ~512 bits each for n=1024)
    n = p * q                                  // public modulus
    φ(n) = (p-1)(q-1)                          // Euler totient
    Choose e: gcd(e, φ(n)) = 1                 // public exponent, often 65537
    d = e^(-1) mod φ(n)                        // private exponent
    Public key:  (n, e)
    Private key: d (or (p, q, d_p, d_q) for CRT speedup)

Enc(m):  c = m^e mod n
Dec(c):  m = c^d mod n     // 由 Euler theorem 保證 m^(ed) ≡ m mod n

Sign(m): σ = m^d mod n
Verify(m, σ): check m == σ^e mod n
```

**正確性 (Euler theorem)**: ed ≡ 1 mod φ(n)，故 ed = 1 + kφ(n)。m^(ed) = m · m^(kφ(n)) = m · (m^φ(n))^k ≡ m · 1^k ≡ m mod n（gcd(m, n) = 1 almost surely）。

**安全性論證**（直觀）：對手有 (n, e, c)。若能 factor n → p, q → φ(n) → d → m，**RSA-break ≤ Factor**。反向 reduction 仍 open（即「能算 RSA 不一定能 factor」）。

## Results
- **40+ 年主導公鑰密碼學**。
- **PKCS#1 (1991+)** 標準化（RSA Labs / IETF）。
- **TLS 1.0-1.2** 用 RSA key exchange + RSA cert signature。TLS 1.3 棄 RSA KEX 但仍允許 RSA cert (用 PSS)。
- **PGP / X.509 / S/MIME** 全用 RSA。
- **2002 Turing Award** 頒給 Rivest, Shamir, Adleman。

## Limitations / what they don't solve
- **沒有 padding scheme**：textbook RSA 直接用會立刻被 multiplicative homomorphism attack 打破 IND-CPA / EUF-CMA。
- **沒有安全性 reduction proof**：1978 年沒有 game-based definition；2000 年代後才補上 RSA-OAEP / RSA-PSS 等 padded version 的證明。
- **計算昂貴**：modular exponentiation with thousand-bit numbers 比 AES 慢 1000+ 倍。
- **Quantum-vulnerable**：Shor 1994 polynomial-time factor。

## How it informs our protocol design
- **G6 不選 RSA**：以下任一理由都足夠：
  - Key/sig size 8× large vs Ed25519。
  - 軟體效能 ~10× slow vs Ed25519。
  - Quantum 過渡期 ML-DSA hybrid 用 EdDSA 更乾淨。
- **G6 必須能 verify RSA-PSS cert**：與 Web PKI 互通的最低要求。
- **G6 educational reference**：RSA padding 失敗史是 protocol-level oracle attack 的經典教訓，深刻影響 G6 設計（不依賴 server-side error message 區分）。

## Open questions
- RSA Problem 與 FACT 的精確 reduction 是否 tight？
- Quantum factor 的 fault-tolerant qubit 需求精確 estimate (Gidney-Ekerå 2021 估 20M physical qubits)。
