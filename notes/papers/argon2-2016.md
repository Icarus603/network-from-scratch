# Argon2: New Generation of Memory-Hard Functions for Password Hashing and Other Applications
**Venue / Year**: IEEE European Symposium on Security and Privacy (EuroS&P) 2016
**Authors**: Alex Biryukov, Daniel Dinu, Dmitry Khovratovich
**Read on**: 2026-05-14 (in lesson 3.3)
**Status**: full PDF (`assets/papers/argon2-2016.pdf`)
**One-line**: Password Hashing Competition (PHC, 2013-2015) 勝出者；用 large-memory + parallel + flexible 參數設計，在 GPU/ASIC 攻擊者前提供 best-known cost asymmetry；OWASP / NIST SP 800-63B 推薦的 modern password hashing function。

## Problem
密碼 hashing 是 cryptographic 設計的特殊問題：使用者密碼 entropy 通常 < 40 bits（"password123" 之類），對手能用 brute-force 試遍。傳統 H(password ‖ salt) 對 GPU/ASIC/FPGA 攻擊極脆弱：一台 RTX 4090 每秒能算 ~10^10 次 SHA-256，1 天內試完所有 8 字元字母組合。需要 function 讓硬體加速沒優勢——**memory-hard function**。

scrypt (Percival 2009) 是第一個 sequential memory-hard，但設計 dated；TMTO (time-memory trade-off) attack 後來顯示 scrypt 並非完美 memory-hard。PHC 2013-2015 公開競賽收 24 候選，2015 年 Argon2 勝出。

## Contribution
1. **三變體**：
   - **Argon2d**: data-dependent memory access。最強 GPU 抵抗，但側信道不安全（memory access pattern 洩 password info）。**密碼學貨幣 PoW 用此**（如 Monero）。
   - **Argon2i**: data-independent memory access。Side-channel safe（同 password 給同 access pattern），但 TMTO 抵抗稍弱。**Server-side password storage 用此**。
   - **Argon2id**: hybrid，前半 i, 後半 d。**OWASP / NIST 推薦的 default**。
2. **可調參數 (m, t, p)**：
   - m: memory cost (KiB)，建議 ≥ 64 MiB (產品環境)。
   - t: time cost (iterations)，建議 ≥ 3。
   - p: parallelism degree，建議 = available CPU threads。
3. **Internal compression based on BLAKE2b**：每 1024-byte block 用 BLAKE2b 變體 G function 處理。
4. **Provable memory-hardness**：對 ASIC 對手，computing Argon2 with time-area complexity TA = Ω(memory · iterations)。
5. **Resistance to TMTO**：Alwen-Blocki 2016 等後續分析證明 Argon2id 比 scrypt TMTO 抵抗更強。

## Method (just enough to reproduce mentally)
**簡化版**：
```text
Inputs: password P, salt S, parameters (m, t, p)

1. Initial hash: H0 = BLAKE2b(P, S, m, t, p, ...)
2. Allocate memory matrix B[p][m/p] of 1024-byte blocks
3. Fill blocks via compression:
    B[i][0] = H0_extend(i, 0)
    B[i][1] = H0_extend(i, 1)
    For j = 2..m/p-1:
        B[i][j] = G(B[i][j-1], B[i'][j'])    // G is BLAKE2b-based compression
        // Argon2d: (i', j') depends on previous block content (data-dependent)
        // Argon2i: (i', j') depends on (P, S, t, m, p, j) only (data-independent)
4. Repeat above for t iterations
5. Output: BLAKE2b of XOR of last column blocks
```

**為什麼 memory-hard 給 GPU/ASIC 困難**：
- GPU/ASIC 強項是 parallel arithmetic，弱項是 memory bandwidth + memory size。
- Argon2 強制每次 G call 讀 1024-byte block from large memory matrix。
- 64 MiB matrix per password attempt → ASIC 要 batch 1000 passwords parallel 需要 64 GB on-chip memory，超過任何商用 ASIC。

## Results
- **PHC 2015 winner**。
- **NIST SP 800-63B (2024 revision)** 推薦 Argon2id。
- **OWASP Password Storage Cheat Sheet** primary recommendation。
- **libsodium、Bouncy Castle、Python passlib** 等 library 內建。
- **WhatsApp 從 PBKDF2 migrate 到 Argon2**（2020+）。

## Limitations / what they don't solve
- 參數選擇 tricky：太低不安全，太高 server 自己慢。建議 100-500 ms per hash on production hardware。
- Argon2d 不適合 server side（side-channel 風險）；Argon2i 對 1-pass 有 TMTO weakness（後續 Argon2i v1.3 修補）。
- 不解決「使用者選 weak password」的根本問題；只能 raise cost factor。

## How it informs our protocol design
- **Proteus 不是 password protocol，但 PSK 模式可選 password derivation**：若 Proteus spec v2 加 PSK from passphrase，必走 Argon2id (m=64 MiB, t=3)。
- **Proteus client TUI 顯示 PSK 強度警告**：即使有 Argon2，weak password 仍 vulnerable to dictionary attack。
- **Proteus 不在 record-layer 用 Argon2**：太慢，不適合 per-record。

## Open questions
- ASIC-resistant proof 在 future advanced ASIC (3D stacking, processing-in-memory) 是否 hold？
- Quantum resistance：Grover 對 password search 提供 quadratic speedup，需要 doubled effective entropy；但 memory-hardness 在量子硬體上的精確 bound 仍 active。
- 與 PAKE 整合（Part 3.9）：用 Argon2 derive PAKE PSK 的最佳 parameter？
